Part 02

Supabase Setup

Create your free database, run the schema, and enable authentication. Takes about 10 minutes.

1
Create a Supabase account and project
  1. Go to supabase.com → click Start your project
  2. Sign up with GitHub (easiest) or email
  3. Click New project
  4. Fill in:
    • Project name: splithouse
    • Database password: choose a strong password and save it somewhere
    • Region: pick the one closest to you (e.g. Southeast Asia for India)
  5. Click Create new project and wait ~2 minutes for it to provision
⚠️
Save your database password — you'll need it if you ever connect directly via a DB client. The anon key (used in the app) is different and shown on the dashboard.
2
Run the database schema

In your Supabase project dashboard: left sidebar → SQL Editor → click New query. Paste the entire SQL below and click Run.

supabase/schema.sql

You should see "Success. No rows returned". All 6 tables are now created.

3
Enable Row Level Security (RLS)

RLS makes sure users can only access their own group's data. Run this second query:

SQL Editor — RLS policies
4
Enable email authentication (magic link)
  1. Left sidebar → Authentication → Providers
  2. Email is already enabled by default — confirm it's on
  3. Make sure "Confirm email" is enabled
💡
Magic link works like this: user enters email → Supabase sends a link → they click it → logged in. No password needed. The session persists in the browser so they only log in once.
5
(Optional) Enable Google login
  1. Authentication → Providers → find Google → toggle it on
  2. You'll need a Google OAuth Client ID and Secret from console.cloud.google.com
  3. Create a project → APIs & Services → Credentials → OAuth 2.0 Client ID
  4. Add your Supabase callback URL as an authorized redirect URI (shown in the Supabase Google provider settings)
⚠️
Google OAuth setup takes an extra 15 minutes. Skip it for now — magic link works great. You can always add it later.
6
Copy your project credentials

Go to: left sidebar → Project Settings → API

You'll see two values — copy both and keep them handy (you'll add them to your app in Part 04):

ValueWhere to find itExample format
Project URL"Project URL" sectionhttps://xxxx.supabase.co
anon public key"Project API Keys" → anon/publiceyJhbGci... (long JWT)
🔒
Never share the "service_role" key — it bypasses all security. You only need the "anon" key in your app. The service_role key should never be in frontend code.
