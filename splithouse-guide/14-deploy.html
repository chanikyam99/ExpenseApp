Part 14

Deploy to Vercel

Push your code to GitHub, connect Vercel, add environment variables, and go live. Takes about 10 minutes.

1
Commit and push all your code to GitHub
Terminal
⚠️
Double-check that .env.local is NOT included in the commit. Run git status and confirm it's not listed.
2
Import the project on Vercel
  1. Go to vercel.com → Dashboard → Add New… → Project
  2. Click Import next to your splithouse GitHub repository
  3. Vercel auto-detects it as a Next.js project — leave all settings as-is
  4. Do not click Deploy yet — add environment variables first
3
Add environment variables in Vercel

On the import screen, expand "Environment Variables" and add both values:

KeyValue
NEXT_PUBLIC_SUPABASE_URLYour Supabase project URL
NEXT_PUBLIC_SUPABASE_ANON_KEYYour Supabase anon key
💡
Set both for all three environments: Production, Preview, and Development. This way, preview deploys (PRs) also work correctly.
4
Click Deploy

Vercel will build and deploy your app. This takes about 60–90 seconds. When done, you'll get a live URL like https://splithouse-abc123.vercel.app.

✅
If the build succeeds, your app is live. Open the URL and try logging in.
5
Add your Vercel URL to Supabase's allowed redirects

Magic-link login requires Supabase to know which URLs are safe to redirect to after login.

  1. Go to your Supabase project → Authentication → URL Configuration
  2. Under "Redirect URLs", add your Vercel URL:
Add both of these to Supabase Redirect URLs
  1. Save. Now magic links will work on the live URL.
6
Set up a custom domain (optional but nice)

In Vercel → your project → Settings → Domains → add a custom domain. If you don't have one, the default .vercel.app URL works fine forever.

7
Invite your housemates
  1. Open your live app and log in
  2. Create your house group (e.g. "Our House 2025")
  3. Go to Members → copy the invite link
  4. Share it in your house WhatsApp group
  5. Each person clicks the link, logs in with their email, enters their name, and joins
🎉
You're live! Add your first real expense and watch the balances update.
8
Future deployments are automatic

Every time you push code to the main branch on GitHub, Vercel automatically rebuilds and redeploys. You don't need to do anything manually.
