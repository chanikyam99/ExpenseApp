Part 04

Config Files

Set up Tailwind, Next.js config, global styles, and your environment variables. These files wire everything together.

tailwind.config.ts

Replace the entire contents of tailwind.config.ts in your project root:

tailwind.config.ts

next.config.ts

Replace next.config.ts (or next.config.mjs) in your project root:

next.config.ts

app/globals.css

Replace app/globals.css with this. It defines CSS variables for the dark color system and scrollbar styling:

app/globals.css

.env.local

Create a file called .env.local in your project root (same level as package.json). Paste your Supabase credentials from Part 02:

.env.local
🔒
Never commit .env.local to git. It is in .gitignore by default. When you deploy to Vercel, you enter these same values in Vercel's dashboard under Project Settings → Environment Variables.

Verify setup

Run the dev server to confirm everything is wired up so far:

Terminal

Open http://localhost:3000. You should see the default Next.js starter page with Tailwind working. We'll replace this with the login page in Part 07.

✅
If the page loads without errors, your configuration is correct. Press Ctrl+C to stop the dev server.
