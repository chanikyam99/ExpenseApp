Verification & Troubleshooting

A test checklist to confirm everything works, plus solutions to every common error you might hit.

Test Checklist

Run each test before sharing the app with housemates. Each is designed to catch a specific failure mode.

1
Magic link login works
  1. Open the app and enter your email
  2. Check your email — you should receive a link within 30 seconds
  3. Click the link — you should land on /groups, logged in
  4. Refresh the page — you should still be logged in (session persisted)
⚠️
If the link redirects to an error, check that http://localhost:3000/auth/callback is in your Supabase Redirect URLs (for local) and https://your-app.vercel.app/auth/callback (for production).
2
Invite link and join flow work
  1. Create a group and copy the invite link
  2. Open it in a private/incognito window
  3. Log in as a different email address
  4. Confirm you land on the join page, enter a display name, and join
  5. Confirm both accounts appear in the Members tab
3
Balance correctness — manual verification

Add 3 expenses with known payers and splits, then verify the balances match your manual calculation.

Example scenario to verify

Expected: Raj owes Ali ₹500 total (after removing Raj's grocery contribution of ₹400 from Ali's coverage). Work it out on paper, then confirm the dashboard shows the same number.

4
Split sum invariant

For every expense, the sum of all splits must equal the total amount exactly. Run this SQL in Supabase SQL Editor:

Supabase SQL Editor
✅
Expected: zero rows. If any rows appear, the equalSplits() function has a bug — check the integer paise arithmetic in lib/balance.ts.
5
Settlement clears the balance
  1. With a non-zero balance showing on the dashboard, click "Record a settlement"
  2. Enter the exact amount owed and submit
  3. Return to the dashboard — balance should show "All settled up!" or a reduced amount
6
Real-time activity feed
  1. Open the Activity page in two browser tabs (both logged in)
  2. In Tab 1, add a new expense
  3. In Tab 2, watch the activity feed — the new entry should appear within 1–2 seconds without refreshing
💡
If real-time doesn't work, go to Supabase → Database → Replication and enable the activity_log table for the supabase_realtime publication.
7
CSV export
  1. Go to the History page
  2. Click "Export CSV"
  3. Open the downloaded file in Excel or Google Sheets
  4. Verify: all expenses appear, columns are correct, amounts match
8
Row-level security — users can't see other groups
  1. Create two separate Supabase accounts (two emails)
  2. Each creates their own group
  3. Log in as User A — confirm they can't see User B's group in the Groups list
  4. Try navigating directly to User B's group URL — confirm it redirects away
9
Mobile experience
  1. Open the live URL on a real phone
  2. Test: login, create group, add expense, check balances
  3. Verify the bottom navigation is reachable with thumbs
  4. Verify forms don't get obscured by the keyboard
  5. In Chrome on Android: tap the "Add to Home Screen" option — the app should install as a PWA

Common Errors & Fixes

🔴
Error: "supabase is not defined" or build fails with module not found
Run npm install again. Make sure @supabase/supabase-js and @supabase/ssr are in package.json.
🔴
Error: "invalid API key" from Supabase
Your .env.local values are wrong or empty. Copy them fresh from Supabase → Settings → API. Restart the dev server after editing .env.local.
🔴
Error: Magic link redirects to "auth_failed"
The callback URL is not in Supabase's allowed list. Go to Supabase → Authentication → URL Configuration → Redirect URLs and add http://localhost:3000/auth/callback and your production URL.
🔴
Error: "relation does not exist" (Postgres error on insert)
The schema SQL was not run. Go to Supabase → SQL Editor and run the full schema from Part 02.
🔴
Error: RLS policy blocks all reads (empty lists everywhere)
The RLS policies were not created or the is_group_member() helper function is missing. Run the RLS SQL block from Part 02 in the SQL Editor. Also confirm the user is in group_members for that group.
🔴
Error: Vercel build fails — "Type error: ..."
TypeScript type errors don't crash the dev server but do block Vercel builds. Run npx tsc --noEmit locally to see all type errors and fix them before pushing.
🔴
Error: "cookies() should be awaited" warning in Next.js 15
This guide targets Next.js 14. If you accidentally installed 15+, either pin to 14 in package.json ("next": "14.2.29") or update lib/supabase/server.ts to use await cookies().
🔴
Real-time feed doesn't update
Enable the table for Realtime: Supabase → Database → Replication → supabase_realtime → toggle on activity_log. Also check the browser console for WebSocket errors.
🔴
Custom splits don't add up — form validation fires
The custom-split form sums all input values and blocks submit if they don't match the total. Check for floating-point input: entering "33.33 + 33.33 + 33.34" is fine, but "33.33 × 3" would show a ₹0.01 discrepancy. Use the equal-split mode for clean splits, or adjust one person's value to absorb the remainder.
🔴
Charts are blank on the Insights page
Recharts requires data with the exact shape expected. Open the browser console — if you see a Recharts prop warning, check that the dataKey props in the BarChart and PieChart match the object keys returned by your aggregation logic.
