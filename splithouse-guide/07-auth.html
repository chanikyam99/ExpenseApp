Part 07

Authentication

The root layout, global styles, login page, and the auth callback handler that completes the magic-link login flow.

app/layout.tsx

Replace the default app/layout.tsx. This is the shell that wraps every page with the correct font and dark background.

app/layout.tsx

app/page.tsx — Login page

Replace the default app/page.tsx. This is the first page users see. It offers both magic-link email and Google login.

app/page.tsx

app/auth/callback/route.ts

When a user clicks the magic link in their email, Supabase redirects them to /auth/callback?code=.... This route handler exchanges the code for a session, then redirects to /groups.

app/auth/callback/route.ts
⚠️
Add callback URL in Supabase: Go to Authentication → URL Configuration → add http://localhost:3000/auth/callback to the "Redirect URLs" list. When deployed, also add your Vercel URL: https://your-app.vercel.app/auth/callback

Test it

Run npm run dev, open http://localhost:3000, enter your email and click "Send magic link". Check your inbox, click the link — you should land on /groups.

✅
Login works when: you enter email → get an email within 30 seconds → click the link → see /groups (even if it's blank for now).
