Part 06

Supabase Clients

Three files that wire Supabase into Next.js — one for the browser, one for server-side code, and the middleware that protects every route.

💡
Why two clients? In Next.js, some code runs in the browser (client components) and some runs on the server (server components, API routes). Each environment handles cookies differently, so Supabase provides two separate helpers.

lib/supabase/client.ts

Used in all 'use client' components (forms, real-time pages, etc.).

lib/supabase/client.ts

lib/supabase/server.ts

Used in server components (pages that fetch data before rendering) and API route handlers.

lib/supabase/server.ts

middleware.ts

Create this file in your project root (same level as package.json). It intercepts every request and redirects unauthenticated users to the login page — except for public routes like login, auth callback, and group invite links.

middleware.ts
💡
How it works: The middleware runs before every page load. It checks if Supabase has a valid session cookie. If not — and the route isn't public — it redirects to / (the login page). This means every protected page is automatically secured.
